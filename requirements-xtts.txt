# Isolated environment for the Polish XTTS-v2 engine (.venv-xtts).
#
# coqui-tts 0.27.x calls transformers.pytorch_utils.isin_mps_friendly, which was
# removed in transformers 5.x — so XTTS must stay on the 4.x line. The main app
# venv keeps transformers 5.x for mlx-audio/Kokoro. The two never share a process.
#
# The "cpu" extra pulls the plain PyPI torch wheel, which is MPS-capable on macOS.
# Pin torch < 2.9: from 2.9 coqui-tts requires torchcodec, which links against
# FFmpeg 4-7 and would break against the FFmpeg 8 on this machine. On torch 2.8
# coqui falls back to torchaudio for IO. No CUDA on Mac, so "cpu" is moot anyway.
coqui-tts[cpu]>=0.27
torch>=2.2,<2.9
torchaudio>=2.2,<2.9
transformers>=4.57,<5
numpy<2
soundfile>=0.12
